4000 Weeks

An Android app for mindful memory-keeping, one week at a time.

Project Overview

"4000 Weeks" is a native Android application designed to reframe the concept of memory-keeping. In an age of cluttered digital photo galleries, this app encourages a more intentional approach by allowing users to save only one meaningful photo per week of their life. This constraint transforms a user's timeline from a random collection of images into a curated, week-by-week visual biography, helping them reflect on their most significant moments.

Key Features

  • Secure User Accounts: Full user registration and login system built with Firebase Authentication, ensuring each user's timeline is private and secure.
  • Weekly Photo Upload: A focused interface for selecting and uploading a single image for a specific week, complete with a comment to add context to the memory.
  • Expandable Visual Timeline: The core UI is a nested, expandable list that organizes a user's life by year, then month, then week, allowing for easy navigation through their entire life timeline.
  • Cloud-Based Backend: Utilizes Firebase for a complete backend solution, with Cloud Storage for robust image file hosting and the Realtime Database for storing image URLs and user comments.

My Role & Contributions

As the sole developer of this personal project, I was responsible for the entire development lifecycle, from initial concept and design to implementation and testing. This involved architecting the complete application in Java using Android Studio.

  • Designed and implemented the full user interface in XML, including the complex, multi-level nested list for the timeline.
  • Engineered all client-side logic in Java for user authentication, image selection from the device gallery, and data handling.
  • Integrated the full suite of Firebase services, including setting up secure rules for both the Realtime Database and Cloud Storage.
  • Conducted iterative testing and debugging to ensure application stability and a smooth user experience.
